VectorBlox MXP Preview: Supported Xilinx Development Boards

Pre-built FPGA bitstreams of VectorBlox MXP systems are available for the development boards listed here.

The bitstreams are for evaluation purposes only. VectorBlox licenses the MXP processor as a soft-core for use in FPGA designs.

Please contact us at if you would like to evaluate the VectorBlox MXP processor on a development board not listed here.

1 Avnet ZedBoard

The ZedBoard contains a Xilinx Zynq-7000 XC7Z020 FPGA with two integrated ARM Cortex-A9 processors running at 667 MHz, and 512 MB of on-board DDR3 DRAM.

For more information on the board, see the following links:

  • ZedBoard website: http://zedboard.org
  • Digilent website (academic pricing): http://www.digilentinc.com/Products/Detail.cfm?NavPath=2,400,1028&Prod=ZEDBOARD

We provide pre-built bitstreams for the ZedBoard for various sizes of the MXP processor (1 to 16 vector lanes) connected to the Zynq's ARM Cortex-A9 processing subsystem.

Note that these bitstreams do not include support for the ZedBoard's on-board HDMI output. If HDMI input and/or output is required, we also have bitstreams that support the on-board HDMI output as well as bitstreams that support the Avnet FMC-IMAGEON daughter-card (see below).

A standalone BSP is included for each system.

The bitstreams and BSPs were created with ISE 14.7.

NOTE: We have tested with ZedBoard Rev C and Rev D boards. We have a report of an incompatibility with a pre-production Rev B board.

The MXP processor in each system is configured as follows:

  • 110 MHz.
  • 64 KB of scratchpad memory.
  • 64-bit wide DMA interface (32 bits for V1 configuration).
  • Fractional bits for fixed-point multiplication: 16 fractional bits for words, 15 for halfwords, 4 for bytes.

2 Avnet ZedBoard with 1080p HDMI Video Output

These bitstreams make use of the ZedBoard's HDMI output port. They consist of:

  • An MXP processor (8, 16 vector lanes) connected to the Zynq's ARM Cortex-A9 processing subsystem.

  • An output video pipeline to read RGB pixels from frame buffers in DRAM and output 1920x1080p @ 60Hz HDMI video.

The output video pipeline is taken from Analog Devices Inc.'s ADV7511 HDMI transmitter reference design for the ZedBoard (http://wiki.analog.com/resources/fpga/xilinx/kc705/adv7511).

A standalone BSP is included for each system.

The bitstreams and BSPs were created with ISE 14.7.

The demonstration program in software/demo/zedboard shows how the MXP processor can be used to generate video frames in these systems. It makes use of Analog Devices Inc.'s ADV7511 Transmitter Library.

The MXP processor in each system is configured as follows:

  • 100 MHz.
  • 64 KB of scratchpad memory.
  • 64-bit wide DMA interface (32 bits for V1 configuration).
  • Fractional bits for fixed-point multiplication: 16 fractional bits for words, 15 for halfwords, 4 for bytes.
  • A custom instruction for accelerating the particle repulsion demo program.

In both the V8 and V16 systems, the custom instruction pipeline is four logical lanes wide (vci_lanes=4).

See the section below for more information on custom instructions.

3 Avnet ZedBoard with FMC-IMAGEON HDMI I/O FMC Module

This setup consists of a ZedBoard (see above) and an Avnet FMC-IMAGEON daughtercard used for HDMI video input and output.

For more information on the FMC-IMAGEON card, see the following links:

  • Avnet FMC-IMAGEON HDMI I/O FMC Module: http://www.em.avnet.com/fmc-imageon
  • The FMC-IMAGEON module can also be bought as a bundle with an ON Semiconductor VITA image sensor (http://www.em.avnet.com/fmc-imageon-v2000c) or as part of the the Xilinx Zynq-7000 SoC Video and Imaging Kit (http://www.xilinx.com/products/boards-and-kits/DK-Z7-VIDEO-G.htm).

We provide pre-built bitstreams for ZedBoard + FMC-IMAGEON systems that consist of:

  • An MXP processor (8, 16 vector lanes) connected to the Zynq's ARM Cortex-A9 processing subsystem.

  • An input video pipeline to store incoming 1920x1080p @ 60Hz HDMI video frames into frame buffers in external DRAM (as RGB pixels).

  • An output video pipeline to read RGB pixels from frame buffers in DRAM and output 1920x1080p @ 60Hz HDMI video.

NOTE: The bitstreams use a time-limited evaluation version of Xilinx's Chroma Resampler IP core (v_cresample) for conversion between YCbCr 4:2:2 and YCbCr 4:4:4 formats, so the video pipelines will stop working (the output will go blank) after about an hour. You will need to power cycle the board before reloading the bitstream.

A standalone BSP is included for each system.

The bitstreams and BSPs were created with ISE 14.7.

The demonstration program in software/demo/zedboard_imageon shows how the MXP processor can be used to process video frames in these systems.

The MXP processor in each system is configured as follows:

  • 100 MHz.
  • 64 KB of scratchpad memory.
  • 64-bit wide DMA interface (32 bits for V1 configuration).
  • Fractional bits for fixed-point multiplication: 16 fractional bits for words, 15 for halfwords, 4 for bytes.
  • A custom instruction for accelerating the particle repulsion demo program.

In the V8 system, the custom instruction pipeline is two logical lanes wide (vci_lanes=2), and in the V16 system it is four logical lanes wide (vci_lanes=4).

See the section below for more information on custom instructions.

4 Avnet ZedBoard MicroBlaze System

These ZedBoard bitstreams consist of an MXP processor connected to a MicroBlaze soft-CPU implemented in the Programmable Logic portion of the Zynq FPGA.

Note that these bitstreams do not include support for the ZedBoard's on-board HDMI output. If HDMI input and/or output is required, we also have MicroBlaze-based bitstreams that support the on-board HDMI output as well as bitstreams that support the Avnet FMC-IMAGEON daughter-card (see below).

A standalone BSP is included for each system.

The bitstreams and BSPs were created with ISE 14.7.

The MicroBlaze processor in each system is configured as follows:

  • 100 MHz.
  • 4 KB data cache, direct-mapped, write-back, 32-byte line size.
  • 8 KB instruction cache, direct-mapped, 32-byte line size.
  • 1024-entry branch target cache.
  • barrel shifter.
  • hardware integer multiplier for mul, muli support.
  • machine status register bit set/clear instructions.
  • pattern compare instructions.
  • reversed load/store and swap instructions.
  • no integer divider, no floating point unit, no memory management unit.

The MXP processor in each system is configured as follows:

  • 100 MHz.
  • 64 KB of scratchpad memory.
  • 64-bit wide DMA interface (32 bits for V1 configuration).
  • Fractional bits for fixed-point multiplication: 16 fractional bits for words, 15 for halfwords, 4 for bytes.

The Xilinx Platform Studio Bus Interfaces and Addresses views are shown below:

Note that DRAM is aliased to the range 0x8000_0000 to 0x9fff_ffff for uncached access via the MicroBlaze's M_AXI_DP port.

7 Custom Instruction Descriptions

As noted above, certain board configurations come equipped with custom instructions such as fixed-point divide, square root, and particle repulsion force calculation. As these custom instructions are specific to particular systems, code written using custom instructions will not be portable to all systems. Systems with the same custom instructions have a variable number of custom instruction lanes, which can affect performance and programming. See the MXP Programming Reference for further information on custom instructions.

7.1 Particle Repulsion

The particle repulsion custom instruction is for accelerating the particle repulsion demo program. The pipeline is described in the following conference paper: A. Severance, J. Edwards, H. Omidian, G. Lemieux, "Soft Vector Processors with Streaming Pipelines," International Symposium on Field-Programmable Gate Arrays (FPGA), February, 2014 (http://www.ece.ubc.ca/~lemieux/publications/severance-fpga2014.pdf).

The particle repulsion custom instruction interface is not standardized, and should only be used in the supplied particle repulsion demo.
